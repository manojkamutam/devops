  • II. Core Ansible Concepts

    • 4. Modules

      Understanding Ansible modules (core and community), using modules in playbooks. Example:

      # Example of a simple playbook (playbook.yml)
---
- name: Install and start Apache
  hosts: webservers
  become: true  # Run tasks with elevated privileges

  tasks:
    - name: Install Apache package
      ansible.builtin.apt:
        name: "{{ package_name }}"
        state: present
        update_cache: true
      vars:
        package_name: apache2

    - name: Start Apache service
      ansible.builtin.service:
        name: "{{ package_name }}"
        state: started
        enabled: true

    • 8. Handlers

      Special tasks that are triggered by other tasks only when they report a change. Example:

      - name: Configure Apache
  ansible.builtin.template:
    src: templates/httpd.conf.j2
    dest: /etc/apache2/apache2.conf
    owner: root
    group: root
    mode: '0644'
  notify: Restart Apache

handlers:
  - name: Restart Apache
    ansible.builtin.service:
      name: apache2
      state: restarted

    • 10. Facts

      Gathering information about managed nodes. Example (accessing facts in a playbook):

      - name: Display system information
  hosts: all
  gather_facts: true

  tasks:
    - name: Show operating system
      ansible.builtin.debug:
        msg: "Operating System: {{ ansible_distribution }} {{ ansible_distribution_version }}"

    - name: Show total memory
      ansible.builtin.debug:
        msg: "Total Memory: {{ ansible_memtotal_mb }} MB"

  • III. Ansible for DevOps Tasks

    • 11. Configuration Management

      Ensuring systems are in a desired state. Examples (as shown in previous sections).

    • 12. Application Deployment

      Deploying and managing applications. Example (simple web app deployment):

      - name: Deploy web application
  hosts: webservers
  become: true

  tasks:
    - name: Create web directory
      ansible.builtin.file:
        path: /var/www/myapp
        state: directory
        owner: www-data
        group: www-data
        mode: '0755'

    - name: Copy application files
      ansible.builtin.copy:
        src: /path/to/your/app
        dest: /var/www/myapp
        owner: www-data
        group: www-data
        mode: '0644'

    - name: Restart web service
      ansible.builtin.service:
        name: apache2
        state: restarted

    • 14. Cloud Provisioning with Ansible

      Interacting with cloud providers (AWS, Azure, GCP) using Ansible modules. Example (AWS EC2):

      - name: Launch an EC2 instance
  hosts: localhost
  gather_facts: false

  tasks:
    - name: Launch EC2 instance
      amazon.aws.ec2:
        key_name: my_keypair
        instance_type: t2.micro
        image: ami-xxxxxxxxxxxxxxxxx
        region: us-east-1
        security_group: my_security_group
        count: 1
        wait: yes
      register: ec2_instance

    - name: Add new instance to hostvars
      ansible.builtin.add_host:
        hostname: "{{ item.public_ip }}"
        groups: launched_ec2
        ansible_user: ubuntu
        ansible_private_key_file: ~/.ssh/my_private_key
      loop: "{{ ec2_instance.instances }}"

    - name: Wait for SSH
      ansible.builtin.wait_for:
        host: "{{ item.public_ip }}"
        port: 22
        state: started
      loop: "{{ ec2_instance.instances }}"
